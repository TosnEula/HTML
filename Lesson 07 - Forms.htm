Form Structure

Using the element form starts the creation of the form.

  • The attribute action is the url for the page that will recieve the information.
  • The method attribute will either "get" or "post" the response
    • The "get" method is useful for:
      1. short forms (such as search boxes)
      2. when you are just retrieving data from the web server (not sending information that should be added to or deleted from a database)
    • The "post" method is useful for:
      1. allows users to upload a file.
      2. is very long.
      3. contains sensitive data (e.g. passwords).
      4. adds information to, or deletes information from, a database.
  • The attribute id is used to identify the form and is used by scripts to verify information.

This is where the form controls will appear.

Text Input

The element input is used to indicate that the end user will need to provide information.

  • type="text" is the attribute used to create a single line text box.
  • The name attribute identifies the individual box, such as username of password.
  • size is used to indicate the amount of characters the box can hold. NO LONGER USED.
  • maxlength, this attribute limits the amount of characters a user can put into the field.

Labelling Form Controls

each form control should have its own label element as this makes the form accessible to vision-impaired users. Ways to use the label attribute includes:

  • Wrap around both the text description and the form input
  • Be kept separate from the form control and use the for attribute to indicate which form control it is a label for

The for attribute states which form control the label belongs to.NOTE for uses the id to indicate its reference.
